# Explainer Shot List

A 4-shot narrative arc designed for explaining concepts, processes, or features.

**Total Duration**: ~26 seconds
**Estimated Cost**: ~$2.00

---

## Narrative Arc

```
PROBLEM → INSIGHT → SOLUTION → OUTCOME
```

| Beat | Purpose | Emotional Goal |
|------|---------|----------------|
| Problem | Establish the challenge or pain point | Recognition, tension |
| Insight | Reveal the key understanding | "Aha", clarity |
| Solution | Show how it's solved | Relief, confidence |
| Outcome | Demonstrate the positive result | Satisfaction, desire |

---

## Design Principles for Explainers

### Abstract Over Literal
Veo excels at abstract visualization, not realistic scenarios. Instead of:
- "Person frustrated at computer" → Abstract tension visualization
- "Code running on screen" → Data flowing through abstract systems
- "Happy customer using product" → Resolved, harmonious abstract forms

### Visual Metaphor Thinking
Translate concepts into visual language:

| Concept | Visual Metaphor |
|---------|-----------------|
| Complexity/Chaos | Tangled forms, scattered particles |
| Simplicity/Clarity | Clean geometry, aligned elements |
| Speed/Efficiency | Flowing streams, smooth motion |
| Security/Safety | Solid structures, containment |
| Growth/Progress | Expanding forms, upward motion |
| Connection | Linking elements, converging paths |

### Emotional Arc Through Visuals
The visual treatment should mirror the emotional journey:

| Beat | Visual Feeling |
|------|----------------|
| Problem | Tension, complexity, friction |
| Insight | Clarity emerging, focus |
| Solution | Resolution, smoothness, order |
| Outcome | Expansion, satisfaction, brightness |

## Shot Specifications

### Shot 1: PROBLEM (6s)

**Purpose**: Visualize the challenge, complexity, or pain point.

| Attribute | Specification |
|-----------|---------------|
| Beat | Problem |
| Duration | 6 seconds |
| Shot Size | Medium or Wide |
| Camera | Static or slow, constrained movement |
| Subject | Abstract representation of the problem |
| Action | Tension, complexity, struggle |

**Emotional Journey**: Recognition → Tension → "This is hard"

**Prompt Elements**:
- Visual metaphor for the problem
- Tension or complexity in forms
- Constrained, difficult feeling
- Colors and light suggesting challenge

**Example Prompt Structure**:
```
[Static or constrained camera], [abstract problem visualization],
[tension/complexity in forms], [challenging atmosphere],
[Visual DNA: palette with tension adjustment], [harder/harsher lighting],
[atmosphere with friction], problem state, [challenge descriptor]
```

---

### Shot 2: INSIGHT (6s)

**Purpose**: The moment of understanding—the key realization.

| Attribute | Specification |
|-----------|---------------|
| Beat | Insight |
| Duration | 6 seconds |
| Shot Size | Medium, focusing |
| Camera | Push in or focus shift |
| Subject | Clarity emerging from complexity |
| Action | Revealing, focusing, understanding |

**Emotional Journey**: Curiosity → Revelation → "I see it now"

**Prompt Elements**:
- Clarity emerging from previous complexity
- Light as metaphor for understanding
- Focus on the key element
- Transition from tension to relief beginning

**Example Prompt Structure**:
```
[Push in or focus-revealing camera], [clarity emerging from complexity],
[light revealing understanding], [focus on key insight element],
[Visual DNA: palette], [lighting transitioning to clarity],
[atmosphere shifting], moment of understanding, [insight descriptor]
```

---

### Shot 3: SOLUTION (8s)

**Purpose**: Show the solution in action—how it works, what it does.

| Attribute | Specification |
|-----------|---------------|
| Beat | Solution |
| Duration | 8 seconds |
| Shot Size | Medium (solution clearly visible) |
| Camera | Orbit, tracking, or demonstrating movement |
| Subject | Solution in action |
| Action | Working, solving, organizing |

**Emotional Journey**: Confidence → Relief → "This solves it"

**Prompt Elements**:
- Solution clearly visualized
- Action showing it working
- Order emerging from previous chaos
- Smooth, confident motion

**Example Prompt Structure**:
```
[Demonstrating camera movement], [solution visualization clearly working],
[action showing problem being resolved], [order and clarity],
[Visual DNA: palette], [confident lighting], [resolved atmosphere],
solution in action, [effectiveness descriptor]
```

---

### Shot 4: OUTCOME (6s)

**Purpose**: The positive result—what success looks like.

| Attribute | Specification |
|-----------|---------------|
| Beat | Outcome |
| Duration | 6 seconds |
| Shot Size | Wide or expanding |
| Camera | Pull out, crane up, or expansive movement |
| Subject | Resolved state, positive result |
| Action | Flourishing, succeeding, thriving |

**Emotional Journey**: Satisfaction → Aspiration → "I want that"

**Prompt Elements**:
- Positive, resolved visual state
- Expansion and possibility
- Brightness and warmth
- Aspirational framing

**Example Prompt Structure**:
```
[Expansive camera movement], [resolved positive state],
[flourishing/success visualization], [expanded possibility],
[Visual DNA: palette warmed/brightened], [optimistic lighting],
[atmosphere of success], positive outcome, [aspiration descriptor]
```

## Complete Example: Data Integration Platform Explainer

**Product**: API integration platform
**Visual DNA**: tech-saas (cool blues, soft diffused light, ethereal tech atmosphere)

### Shot Table

| # | Beat | Duration | Camera | Subject | Key Moment |
|---|------|----------|--------|---------|------------|
| 1 | Problem | 6s | Static | Tangled data streams | Complexity |
| 2 | Insight | 6s | Push in | Single clear connection | Clarity |
| 3 | Solution | 8s | Orbit | Organized data flowing | It works |
| 4 | Outcome | 6s | Pull out | Harmonious data ecosystem | Success |

### Full Prompts

**Shot 1 - Problem**:
```
Static camera observing tangled abstract data streams crossing chaotically,
multiple conflicting pathways creating visual complexity, friction points glowing,
cool blue palette with harsh red-orange tension accents, harder diffused light,
tech atmosphere with friction, data chaos, the problem of disconnection
```

**Shot 2 - Insight**:
```
Smooth push in as single clear data connection illuminates from the tangle,
clarity emerging, light revealing the path forward, focus on key integration point,
cool blue palette, lighting transitioning from harsh to clear,
ethereal tech atmosphere lifting, moment of understanding, the insight
```

**Shot 3 - Solution**:
```
Smooth orbit around organized data hub, streams flowing cleanly through center,
abstract API connections working harmoniously, data moving with purpose,
cool blue palette with soft warm success accents, confident soft lighting,
ethereal tech atmosphere, solution in action, seamless integration working
```

**Shot 4 - Outcome**:
```
Smooth pull out revealing vast harmonious data ecosystem,
multiple integrations flowing perfectly, expansion and possibility,
cool blue palette brightened with warm success accents, optimistic ambient light,
ethereal tech atmosphere elevated, flourishing connected state, success achieved
```

## Customization Guide

### Adjusting Complexity Level

| Audience | Modifications |
|----------|---------------|
| Technical | More detailed visualizations, faster pacing |
| General | Simpler metaphors, clearer transformations |
| Executive | More abstract, emphasis on outcomes |

### Alternative Narrative Structures

| Variation | Arc | When to Use |
|-----------|-----|-------------|
| Feature-focused | Context → Feature → Benefit → Result | Product features |
| Process-focused | Before → Step 1 → Step 2 → After | Process explanation |
| Comparison | Old Way → Problem → New Way → Better | Competitive positioning |

### Duration Variations

| Total Target | Shot Durations | Best For |
|--------------|----------------|----------|
| ~20s (shorter) | 4, 4, 6, 6 | Quick concept intro |
| ~26s (default) | 6, 6, 8, 6 | Standard explainer |
| ~32s (extended) | 8, 8, 8, 8 | Complex concept |

---

## Transitions

**Recommended**: Crossfade (0.5s)
- Smooth conceptual transitions
- Transformation feeling between states
- Professional presentation style

**Alternative**: Cut
- Use between Problem→Insight (abrupt clarity)
- Creates contrast between states
- More energetic feel

---

## Post-Production Notes

Explainer videos typically need:
- **Voiceover** narrating the concept (add in post)
- **Text labels** for key concepts (add in post)
- **Logo/branding** at end (add as end card)
- **Music** supporting the emotional arc (add in post)

Generate clips without audio, add VO and music in post for timing control.
